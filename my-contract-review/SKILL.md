# my-contract-review — 계약서 검토 스킬

계약서를 4단계 프레임워크로 검토하는 스킬.
PDF/DOCX/텍스트 파일을 읽고, EO 계약 정책 + 한국 법률 + 업계 벤치마크 기준으로 분석.

## 트리거

"계약서 검토", "계약 분석", "contract review", "이 계약서 봐줘", "계약 리스크"

## 사용법

```
"이 계약서 검토해줘" + 파일 경로
"계약서 검토해줘 --perspective counterpart" (상대방 입장)
"계약서 검토해줘 --perspective neutral" (중립)
```

기본 입장은 EO 유리(eo).

---

## 검토 프레임워크 (4단계)

### 1단계: 기본 검증
- 금액/날짜/당사자명 정확성 (템플릿 미수정 잔재 확인)
- 필수 조항 존재 여부 (계약기간, 대금, 해지조건)
- 인감/서명란 존재 여부

### 2단계: EO 정책 기준
- 저작권 귀속이 계약 유형에 맞는지
  - 미디어 상품 → 저작권 EO 귀속
  - 프로덕션(납품형) → 저작권 고객사 귀속
  - 출연(무페이) → 저작권/초상권/편집권 EO 귀속
- 완성본/클린본/원본 3단계 구분 명시 여부
  - 완성본: 유튜브 업로드 버전 (비상업적 용도 허용)
  - 클린본: 음악/자막/워터마크 제거 (20~30% 별매)
  - 원본: 별도 계약
- 수정 횟수 제한: 2회 무료, 3회부터 50만원/회(VAT 별도)
- 영상 내림 조건: 출연자/출연사 사회적 물의 시만 내림

### 3단계: BM 고도화 제안
- 업계 비교: CJ ENM, Studio Dragon, 샌드박스, 72초TV 등 계약 관행
- 라이선스 기간 제한 도입 시 갱신 수익 vs 이탈 리스크
- 클린본 가격 적정성 벤치마크
- "현재 EO는 이렇게 → 업계는 이렇게 → 이렇게 바꾸면 이런 효과" 형태

### 4단계: 누락 조항 탐지

**공통 필수 조항:**
- 계약 목적
- 계약 기간 (시작일/종료일)
- 대금 및 지급 조건 (금액, 부가세, 지급 시기)
- 저작권/지식재산권 귀속
- 비밀유지 (기밀정보 범위, 의무 기간)
- 계약 해지/해제 조건
- 손해배상 (범위, 상한)
- 분쟁 해결 (관할 법원 or 중재)

**유형별 추가 필수 조항:**
- 미디어: 수정 횟수 제한, 완성본/클린본/원본 구분, 영상 내림 조건, 2차 저작물 권리
- 프로덕션: 납품 범위/일정, 검수 절차, 하자 보수, 원본 footage 소유권
- 출연: 초상권/퍼블리시티권, 편집권, 재촬영 조건, 클립/숏폼 재활용 범위
- B2G/B2B: 납품 검수 기준, 지체상금, 하도급 조건, 지식재산 사전 양도 동의

## 입장별 검토 (perspective)

### EO 유리 (기본)
- EO에 불리한 조항 우선 식별
- 저작권, 대금, 책임 제한에서 EO 보호 관점
- 수정 제안은 EO 이익 최대화 + 상대방 수용 가능한 현실적 수준 병기

### 상대방 유리 (counterpart)
- 상대방에 불리한 조항 우선 식별
- 납품 범위, 저작권 양도, 면책에서 상대방 보호 관점
- 수정 제안은 상대방이 제안할 법한 방향 + EO 수용 가능 수준 병기

### 중립 (neutral)
- 일방에게 지나치게 유리/불리한 조항 식별
- 양측 리스크 균형 분석
- 양측 모두 수용 가능한 합리적 수정안 제시

---

## 한국 법률 체크포인트

### 약관규제법
- 일방적으로 불리한 조항 → 자동 무효 (제6조)
- 부당한 면책/면책 제한 → 무효 (제7조)
- 과중한 손해배상 의무 → 금지 (제8조)
- 해제/해지권 부당 배제 → 무효 (제9조)

### 하도급법
- 대금 지급기한: 검사 완료 후 60일 이내 (제13조)
- 부당 감액/반품/위탁취소 금지 (제10조, 제11조)
- 기술자료 유용 금지 (제12조의3) — 외주 계약 시 주의
- EO가 원사업자(발주사)인 경우 반드시 준수

### 전자상거래법
- 디지털 콘텐츠: 청약 철회 제한 고지 의무 (제17조)
- 광고/협찬 콘텐츠: 표시광고법 경제적 이해관계 명시
- 해외 플랫폼 수익 분배 시 국제 과세 고려

## 출력 형식

검토 결과는 아래 구조로 출력한다:

**경영진 요약**: 30초 안에 읽을 수 있는 한줄 요약
**전체 리스크**: red(위험) / yellow(주의) / green(안전)

**조항별 분석** (priority 순):
- 리스크 수준 (red/yellow/green)
- 해당 조항 원문 인용
- 문제점
- 수정 제안
- 업계 비교 코멘트
- 확신도 (0~100)
- 우선순위 (1~5, 1=서명 전 반드시 수정)

**누락 조항**: 조항명 + 중요도(필수/권장) + 권장 조항 문구(제X조 형태)

**BM 고도화 제안**: 업계 벤치마크 기반 수익 개선 방안

**후속 조치**: 우선순위별 구체적 액션 (1순위=즉시, 2순위=협의, 3순위=장기)

## 계약 유형 자동 판별

파일 내용에서 아래 키워드로 계약 유형을 자동 판별:
- "영상 제작", "콘텐츠 제작", "유튜브" → 미디어
- "납품", "용역", "위탁" → 프로덕션
- "출연", "촬영", "인터뷰" → 출연
- "교육", "강연", "운영대행" → B2G/B2B
- "광고", "배너", "eDM" → 광고

판별이 모호하면 사용자에게 확인 후 진행.

## Airtable 정책 DB 연동 (선택)

Airtable MCP가 활성화되어 있으면 계약정책 DB에서 최신 정책을 가져와 검토에 반영한다.
- Base: `apphaMgCZMSN3ysHk` (이오테이블)
- Table: `tblmQCBM3ac8Iqcjw` (계약정책)
- Filter: `{IsActive}=TRUE()`
- 정책이 있으면 시스템 프롬프트에 동적 주입
- 없으면 이 문서의 하드코딩 정책으로 fallback

## 조언 원칙

1. **YES/NO 먼저, 조건은 뒤에** — "이 조항은 문제가 있습니다" 먼저, "왜냐하면..." 뒤에
2. **리스크 정량화** — "위험합니다"(X) → "최대 3천만원 위약금 가능"(O)
3. **변호사 확인 경계 명시** — AI 분석의 한계를 명확히 표시
4. **액션 중심** — "문제입니다"(X) → "제5조를 이렇게 수정하세요"(O)

## 주의사항

- 이 스킬은 법률 자문이 아닙니다. 최종 판단은 변호사 확인 필요
- 민감한 계약서 내용은 Claude API로 전송됩니다 (Anthropic 데이터 정책 적용)
- 계약 정책은 Airtable에서 관리하며, 정책 변경 시 자동 반영
- Lisa(법률 어드바이저) 페르소나와 함께 사용하면 더 깊은 분석 가능: `/consult 법률`
