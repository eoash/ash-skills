# EO 디자인 Self-Grading 프롬프트 템플릿

> 출처: Metics Media — "Build $10,000 Websites using Claude Code" (2026-05-29)의 8-pillar 체크리스트 + self-grading 루프를
> **EO 공통 디자인 시스템**(다크 `#0A0A0A` / 액센트 `#00E87A` / Next.js + Tailwind)에 맞게 구체화한 버전.
> 용도: contract-review · finance-dashboard · eo-portal · token-dashboard 등 **EO 내부 웹 도구**의 한 화면을
> "그냥 작동하는" 수준에서 "비싸 보이고 빠르게 느껴지는" 수준으로 끌어올린다.
>
> 핵심 철학(영상에서 가져온 3가지):
> 1. **기준을 먼저 못 박는다** — 만들기 전에 8기둥을 정의하고, 빌드 후 그 기준으로 채점한다.
> 2. **"무엇을 추가하라"가 아니라 "어떻게 느끼게 하라"** — intent over specifics. Claude는 명세화보다 의도 구체화를 잘한다.
> 3. **하나씩 말고 batch로** — 작은 수정 5개를 한 프롬프트에. 토큰 절약 + 결과 응집력.

---

## 0. EO 디자인 시스템 (레퍼런스 = 모든 채점의 기준선)

| 토큰 | 값 | 규칙 |
|------|-----|------|
| 배경 | `#0A0A0A` (near-black) | 기본 캔버스. 순수 검정(`#000`) 금지 — 약간의 온기. |
| 액센트 | `#00E87A` (EO green) | **단 하나의 액센트.** CTA·핵심 지표·활성 상태에만. 남발 금지. |
| 텍스트 | 흰색 계열 1~3단계 위계 (예: `#FAFAFA` / `#A1A1AA` / `#52525B`) | 무지개 팔레트 금지. 절제가 곧 품질. |
| 스택 | Next.js + Tailwind | 차트는 recharts. |
| 폰트 | **Inter 금지** (AI-tell) | 본문은 Geist/Pretendard 등으로. 한글 UI는 Pretendard 권장. |

## 1. 8-Pillar 체크리스트 (EO 내부 도구용으로 구체화)

### 취향(Taste)
1. **관점(Point of View)** — 이 화면이 *누구의 어떤 수작업*을 없애는지 한 문장으로 말할 수 있는가? (예: "운영팀이 4계좌를 수동 대조하던 걸 한 화면에서 끝낸다.") 무드보드가 아니라 방향성.
2. **타이포그래피** — 헤드라인/본문 폰트 페어가 정해져 있는가? **Inter가 보이면 즉시 교체**(AI-tell). 한글+숫자 혼용 시 가독성(tabular-nums) 확보.
3. **색상** — 배경 `#0A0A0A`, 액센트 `#00E87A` 하나만. 액센트가 CTA·핵심 KPI·활성 탭에만 쓰이는가? 경고/위험은 별도 1색까지만.

### 실체(Substance)
4. **위계(Hierarchy)** — 가장 중요한 숫자/액션이 가장 크고, 메타데이터가 가장 작은가? 대시보드라면 KPI > 표 > 타임스탬프 순으로 눈이 흐르는가? 균일 크기 = 평평하고 스캔 불가.
5. **이미지/그래픽** — 내부 도구는 사진 대신 **데이터 시각화·아이콘**이 이미지 역할. 차트가 장식이 아니라 정보 위계를 갖는가? 빈 상태(empty state)에 그래픽이 있는가?
6. **카피** — 절제된 한국어 비즈니스 톤. **em-dash(—) 금지**(AI-tell, 마침표·쉼표·콜론으로). raw JSON·마크다운 아티팩트(`**`)가 plaintext UI에 노출되지 않는가? 통화는 `₩XXX · $YYY` 듀얼 표기.

### 체감 품질(Felt Quality)
7. **모션** — 절제된 마이크로 인터랙션만. 로딩은 스켈레톤, 상태 전환은 부드럽게. 시끄럽거나 과한 애니메이션 금지("more expensive, not busier").
8. **모바일 + 보이지 않는 것** — 반응형 ≠ 모바일 설계(무엇을 숨기고/좁히고/리사이즈할지). 그리고 **felt speed**: API는 401 JSON(302 리다이렉트 금지), 날짜는 KST 유틸(`toISOString` 금지), 낙관적 업데이트로 빠르게 느껴지는가?

## 2. ① Self-Grading 프롬프트 (먼저 채점시킨다 — 붙여넣어 사용)

```
이 화면을 아래 EO 8-pillar 체크리스트로 채점해줘. 각 항목을 강함(Strong) / 혼합(Mixed) / 빠짐(Missing)으로
솔직하게(be honest) 등급 매기고, 왜 그 등급인지 1줄 근거를 달아줘. 점수를 후하게 주지 마.

[EO 디자인 시스템 기준선]
- 배경 #0A0A0A, 액센트 #00E87A 단 하나, 무지개 팔레트 금지
- Inter 폰트 금지(AI-tell), 외부 카피 em-dash 금지, 통화는 ₩·$ 듀얼 표기
- API 미인증은 401 JSON(302 금지), 날짜는 KST 유틸, recharts 차트

[8-pillar]
1. 관점(POV): 누구의 어떤 수작업을 없애는가가 화면에 드러나는가
2. 타이포그래피: 폰트 페어 명확 + Inter 미사용 + 숫자 tabular-nums
3. 색상: #0A0A0A 배경 + #00E87A 액센트가 핵심 요소에만 절제 사용
4. 위계: 가장 중요한 숫자/액션이 가장 크게, 메타가 가장 작게
5. 이미지/그래픽: 차트·아이콘이 정보 위계를 갖고 empty state까지 설계
6. 카피: 절제된 한국어 톤 + em-dash 없음 + raw JSON/마크다운 노출 없음
7. 모션: 절제된 마이크로 인터랙션 + 로딩 스켈레톤, 과한 애니메이션 없음
8. 모바일+felt speed: 모바일 전용 설계 + 401 JSON + KST 날짜 + 빠른 체감속도

마지막에 "가장 손이 필요한 3개 항목"을 우선순위로 뽑아줘. 아직 코드는 고치지 말고 진단만.
```

## 3. ② Batch-Fix 프롬프트 (intent로 한 번에 — 채점 후 사용)

> 핵심: 하나씩 고치지 말 것. **무엇을 추가하라 대신 어떻게 느끼게 하라**로 의도를 먼저 던진다.

```
위 채점에서 Mixed/Missing으로 나온 항목들을 하나씩 말고 한 번에(batch) 고치는 제안을 줘.
구체 명세를 내가 주는 게 아니라 의도를 말할 테니, 네가 구체안으로 번역해줘:

"이 화면이 더 비싸 보였으면 좋겠어. 더 바쁘게 말고, 더 절제되고 정돈되게.
핵심 KPI는 한눈에 들어오고, 나머지는 조용히 뒤로 빠지는 느낌."

EO 디자인 시스템(#0A0A0A / #00E87A 단일 액센트 / Inter 금지)은 절대 벗어나지 마.
제안을 5개 이내로 묶고, 승인하면 한꺼번에 적용해줘.
```

---

## 4. ③ Drive-Yourself 모션 패스 (직접 운전 — 평평한 섹션만)

> Claude는 화면을 "느끼지" 못한다. batch 적용 후 직접 스크롤하며 평평한 섹션을 찾아, **섹션당 하나씩** 지시.

```
[평평한 섹션 이름]이 아직 정적이고 밋밋해. 여기 우아하고 절제된 마이크로 인터랙션 하나만 추가해줘.
EO 액센트(#00E87A)를 과하지 않게 써서. 너무 노골적이면 "더 미묘하게"라고 다시 말할게.
```

(예: KPI 카드 hover 시 액센트 underline reveal / 표 행 hover highlight / 숫자 count-up / 활성 탭 글로우)

## 5. EO 가드레일 (이 프롬프트들과 함께 항상 적용)

- **Inter 보이면 교체** — AI가 만들었다고 즉시 들통나는 폰트. → Geist/Pretendard.
- **em-dash(—) 금지** — 외부 노출 카피의 AI-tell. 마침표·쉼표·콜론으로.
- **액센트는 하나** — `#00E87A`를 여기저기 칠하면 오히려 싸 보인다. CTA·핵심 지표·활성 상태에만.
- **raw JSON / `**` 마크다운 노출 금지** — Claude 응답을 그대로 화면에 뿌리지 말 것. 5단계 방어 파싱 + sanitize.
- **API 미인증 = 401 JSON** — 302 리다이렉트는 클라이언트 `JSON.parse` 크래시.
- **날짜는 KST 유틸** — `toISOString().slice(0,10)`는 UTC라 하루 밀린다. `todayKST()`/`nowKST()`.
- **통화 듀얼 표기** — `₩XXX · $YYY` (FX 1470 기본).
- **정적 vs 백엔드 구분** — 영상의 Hostinger premium↔business 논리는 EO엔 무관(Vercel 사용). 단 "Claude가 React를 거부하면 아키텍처가 복잡해진 신호"라는 교훈은 유효.

## 6. 한 줄 요약 워크플로우

```
① self-grading(진단) → ② batch-fix(의도로 한 번에) → ③ drive-yourself(평평한 섹션 모션 1개씩) → 재채점
```
이 루프를 "전부 Strong"이 될 때까지 2~3회 돌리면, 외주 없이 화면 하나가 expensive해진다.
