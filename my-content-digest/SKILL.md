# my-content-digest

fetch 스킬이 가져온 콘텐츠(트윗, YouTube 등)를 Quiz-First 방식으로 소화하는 스킬.
요약을 먼저 보여주지 않고 퀴즈부터 출제하여 기억 강화 효과를 극대화한다.

## 전체 흐름

```
1. 콘텐츠 받기 (fetch 스킬 결과물 또는 직접 입력)
2. Pre-Quiz (3문제) — 요약 보기 전에 먼저!
3. 선택적 콘텐츠 제공 (결과에 따라 사용자 선택)
4. 본 퀴즈 (9문제: 기본 3 + 중급 3 + 심화 3)
5. 결과 + 후속 선택
```

---

## Pre-Quiz (콘텐츠 보기 전 3문제)

### 핵심 원칙

> **요약을 먼저 보여주지 않는다.**
> 퀴즈에서 틀려야 호기심이 생기고, 그 상태에서 읽으면 기억에 더 잘 남는다. (Pretesting Effect, 기억력 9-12% 향상)

### 출제 방법

1. 콘텐츠에서 핵심 주제 3개 추출
2. 각 주제로 AskUserQuestion 4지선다 문제 출제:
   - 문제 유형: 핵심 개념 / 저자 핵심 주장 / 주요 원칙
   - 각 선택지에 `label` + `description` 포함

### 출제 원칙

| ✅ 출제해야 할 것 | ❌ 출제하지 말 것 |
|-----------------|----------------|
| 핵심 메시지 | 날짜, 연도 |
| 주요 원칙 | 통계 수치 |
| 저자 핵심 주장 | 지엽적 세부사항 |
| 개념 간 관계 | 예시의 세부 내용 |

### 피드백

각 문제 직후 즉시:
- 정답이면: "정답입니다! [간단한 이유]"
- 오답이면: "아쉽게도 오답입니다. 정답은 [N]번 — [이유]. 이 부분을 콘텐츠에서 확인해보세요."

---

## 선택적 콘텐츠 제공

Pre-Quiz 완료 후 AskUserQuestion으로 사용자가 선택한다:

```
옵션 1: "틀린 문제 관련 섹션만"
  → 오답 문제의 답이 담긴 부분만 발췌하여 제공

옵션 2: "핵심 인사이트 3개"
  → 콘텐츠에서 가장 중요한 포인트 3개만 정리

옵션 3: "전체 요약 + 인사이트"
  → 3-5문장 요약 + 인사이트 3개 종합 제공

옵션 4: "바로 본 퀴즈로"
  → 콘텐츠 제공 없이 9문제 바로 시작
```

## 본 퀴즈 (9문제)

### 구조

3난이도 × 3문제 = 총 9문제. 단계별로 AskUserQuestion 사용.

#### 기본 단계 (3문제)

- 핵심 메시지 이해
- 주요 개념 정의
- 주요 원칙 파악

#### 중급 단계 (3문제)

- 개념 간 관계 이해
- 주장의 근거 연결
- 비교·대조 분석

#### 심화 단계 (3문제)

- 실제 상황에 적용
- 핵심 원리 확장
- 비판적 사고 (반례, 한계)

### 출제 원칙

- ✅ 핵심 주제, 원칙, 개념 관계
- ❌ 지엽적 세부사항, 날짜, 통계 수치
- 각 단계 완료 후 즉시 정답 + 해설 제공

### 후속 선택

9문제 완료 후 AskUserQuestion으로 선택:

```
- "다른 문제로 재퀴즈" → 새로운 각도에서 문제 재출제
- "관련 콘텐츠 추천" → 이 주제와 연관된 콘텐츠 3개 추천
- "종료" → 학습 완료
```
